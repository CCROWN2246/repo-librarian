### Phase B — stand up automated ("test the crap out of it") harness
Phase shift Chris explicitly called: from "does it feel right" (HITL, paused) → "prove it works, find every
bug" (objective, agent-drivable). This is a distinct, large phase — a natural point for a FRESH Claude with
a full budget if desired. Three layers, all recommended:
1. **Expand the permanent test suite** — property/determinism/edge tests in CI; every found bug → a
   regression test. The durable proof.
2. **Synthetic-corpus fuzzer** (build on `benchmarks/`): generate random docs-heavy repos, assert invariants
   (below). Headless.
3. **A Workflow that turns agents loose in the sandbox** — one agent per command × input matrix
   (happy/edge/adversarial), asserting invariants (not judging feel), adversarially verifying each failure,
   returning a structured bug list → which becomes new tests.

**Invariants to assert (the test targets):**
- Determinism: run-twice zero diff for `index`/`dream`/`verify` on ANY corpus.
- Exit-code contract: 0 clean / 1 findings / 2 error, uniform across commands.
- Ingest: no-TTY + no `--authority` ALWAYS exit 2 (never silent file); `--dest *.md` errors.
- Apply: idempotent (run-twice zero diff), staleness guard fires, never clobbers, writeback marks applied,
  merge fold idempotent + canonical-guard.
- Verify: DRIFT detection, SKIP contract (skip_if_unset/skip_unless/exit-3), provenance persistence.
- No command CRASHES on: empty repo, malformed frontmatter, malformed `.librarian.toml`, corrupt
  proposals.json, missing files, unicode, huge inputs.
- Round-trip: propose → apply → `todos`/dream reflect state; applied doesn't re-surface.
- Golden stability; `librarian why` reads provenance.
