## Layer 3 result — 11 bugs (6 medium, 5 low), ZERO critical
Full detail + repros + proposed regression tests: **`docs/roadmap/layer3-findings-2026-07-15.md`**.
All are "fail loud / guard the degenerate case" violations — semantic/honesty bugs the
no-crash fuzzer can't find. I re-grounded the mediums against live `src` myself.

**Medium (ranked most-destructive first):**
1. **merge** — self-referential merge (`canonical == redundant`) silently retires the kept
   doc, reports success. (No guard in `_validate_type_specifics` or `_apply_merge`.)
2. **verify** — corrupt/unparseable `baselines.json` → `load_baselines()` returns `{}`
   silently → track checks report NEW not CHANGED → exits green; `--update-baselines` then
   cements the drift. Defeats the headline promise. **This refines the v0.4.1 Bug-4 fix**
   (I made sidecars tolerate non-dict; for baselines that masks drift — must distinguish
   *absent* (fine) from *present-but-corrupt* (warn/fail loud)).
3. **apply (fix)** — `_apply_fix` does `text.replace(old, new, 1)`, ignoring `target.line`;
   a non-unique `old` edits the wrong (frozen) occurrence and reports "applied". **Design
   choice for the fix:** reject a non-unique `old` at propose-validation, OR use `target.line`
   to pick the occurrence at apply.
4. **merge** — a REFUSED merge has already folded the canonical (non-reversible text edit)
   but the outcome + apply-log say only "refused" — non-atomic partial merge misreported as
   a no-op. Fix: check the archive dest BEFORE folding (or disclose the partial state).
5. **ingest** — the existing-frontmatter branch only applies `authority`; `--read-when`/
   `--domain`/`--status`/`--recheck` are silently dropped, success reported, and the
   defaults disclosure names a domain default that wasn't applied.
6. **backfill** — `slug(rel)` id with no collision guard; two paths slugging to the same id
   get duplicate ids in the catalog, no warning (the registry loader guards this — asymmetric).

**Low (5):** query `--json` silent `-n` truncation (no `total`/`truncated` → false-absence);
deletion-fix (`new=""`) false-STALE on the documented re-propose+re-apply route; merge
`carry_over` doesn't dedup *within* a single content list; `why --json` emits empty stdout
on the no-provenance first run (breaks `--json | jq` contract); backfill lifts a title from
a `#` comment inside a fenced code block.

**Verified-CORRECT (do not re-flag):** per-target `_carry_done` guard (no whole-doc-substring
false-fold); cross-op and against-existing `carry_over` dedup; the canonical's intentional
exclusion from the generic pre-handler stale gate (guarded instead inside `_fold_carry_over`).
